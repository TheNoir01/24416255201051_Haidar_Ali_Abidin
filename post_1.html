Apa yang anda harapkan dari mata kuliah basis data?

Posted by Haidar Ali Abidin on September 30, 2025

Halo, para pejuang kode!

Tak terasa, perjalanan saya sebagai mahasiswa Teknik Informatika sudah memasuki semester tiga. Salah satu mata kuliah yang paling sering dibicarakan dan katanya akan menjadi "fondasi" penting untuk karir ke depan adalah Basis Data. Jujur, ada campuran rasa penasaran dan sedikit gentar. Selama ini, kita mungkin hanya menjadi penggunaâ€”mengisi formulir online, login ke media sosial, atau belanja di e-commerce. Tapi, pernahkah kita bertanya: ke mana semua data itu pergi? Bagaimana data miliaran pengguna bisa diatur sedemikian rupa sehingga bisa diakses dalam hitungan detik?

Nah, di semester inilah saya berharap akan menemukan jawabannya. Melalui postingan ini, saya ingin berbagi ekspektasi dan harapan pribadi saya terhadap mata kuliah Basis Data. Anggap saja ini sebagai "kontrak belajar" untuk diri sendiri.

Apa saja harapan dari saya?

Setelah sedikit mencari tahu dan mendengar cerita dari para senior, berikut adalah beberapa hal yang sangat saya harapkan bisa dipelajari dan dikuasai dari mata kuliah ini:

1. Memahami Fondasi Paling Dasar: Dari Data Menjadi Informasi

Harapan pertama tentu saja adalah memahami konsep fundamentalnya. Bukan sekadar hafal definisi, tapi benar-benar mengerti:

  • Apa itu data, basis data (database), dan Sistem Manajemen Basis Data (DBMS)?
  • Mengapa kita butuh database? Kenapa tidak disimpan di Excel saja? (Ini pertanyaan serius!)
  • Bagaimana evolusi database dari model hierarkis hingga relasional yang kita gunakan sekarang?

Saya ingin membangun fondasi yang kokoh, karena tanpa pemahaman konsep, belajar hal teknis seperti menulis query hanya akan terasa seperti menghafal mantra tanpa tahu artinya.

2. Menguasai Bahasa untuk berbicara dengan data(SQL)

Ini mungkin bagian yang paling ditunggu-tunggu. Saya sangat ingin fasih menggunakan SQL (Structured Query Language). Harapan saya tidak muluk-muluk, tapi cukup fungsional untuk bisa:

  • Membuat dan Mengelola Data (CREATE, INSERT, UPDATE, DELETE): Tahu cara menambahkan data baru, mengubahnya jika ada kesalahan, dan menghapusnya saat tidak diperlukan.
  • Mengambil Informasi yang Tepat (SELECT): Ini yang paling krusial. Saya ingin bisa menarik data spesifik dari tumpukan data yang besar. Misalnya, "Tampilkan semua mahasiswa yang mengambil mata kuliah X dan memiliki IPK di atas 3.5."
  • Menggabungkan Data dari Berbagai Tabel (JOIN): Saya dengar ini adalah salah satu skill paling kuat di SQL. Memahami cara menghubungkan tabel data pelanggan dengan tabel data pesanan adalah kunci untuk mendapatkan insight yang berharga.

3. Menjadi Arsitek Data: Merancang Skema Database yang Efisien

Katanya, membuat aplikasi itu seperti membangun rumah. Jika database adalah fondasinya, maka skema atau desain database adalah cetak biru (blueprint)-nya. Saya berharap bisa belajar:

  • Entity-Relationship Diagram (ERD): Mampu merancang ERD untuk memvisualisasikan bagaimana berbagai entitas (seperti Mahasiswa, Mata Kuliah, Dosen) saling berhubungan.
  • Normalisasi: Mengerti mengapa struktur tabel harus dipecah-pecah (dinormalisasi) untuk menghindari redundansi data dan menjaga integritasnya. Saya ingin paham bedanya 1NF, 2NF, 3NF dalam praktik, bukan hanya teori.

Sebuah database yang dirancang dengan buruk akan menyebabkan aplikasi yang lambat dan sulit dikelola di kemudian hari. Saya tidak mau itu terjadi pada proyek saya nanti.

4. Menghubungkan Teori dengan Dunia Nyata

Tentu saja, semua teori ini tidak akan ada artinya jika tidak bisa diimplementasikan. Harapan terbesar saya adalah bisa menghubungkan database yang saya buat dengan aplikasi sederhana. Entah itu website dengan PHP/Python, atau aplikasi desktop sederhana. Saya ingin merasakan sendiri bagaimana sebuah aplikasi "berkomunikasi" dengan database untuk menyimpan dan mengambil data.

Ini adalah momen "aha!" yang saya cari, di mana titik-titik antara mata kuliah Algoritma & Pemrograman dengan Basis Data akhirnya terhubung.

Bukan Sekadar Lulus, Tapi Membangun Bekal

Pada akhirnya, harapan saya dari mata kuliah Basis Data bukan hanya untuk mendapatkan nilai A. Saya melihat ini sebagai investasi skill yang fundamental. Entah nanti karir saya akan menjadi Backend Developer, Data Analyst, Data Scientist, atau bahkan Game Developer, pemahaman tentang cara kerja data akan selalu relevan.

Saya ingin keluar dari kelas ini dengan kemampuan untuk melihat sebuah sistem dan bisa membayangkan "Oh, di belakangnya mungkin ada tabel user, tabel produk, dan tabel transaksi yang saling terhubung."

Bagaimana dengan kalian? Terutama teman-teman satu angkatan atau para senior, apa harapan dan pengalaman kalian dengan mata kuliah Basis Data? Bagikan di kolom komentar ya!
